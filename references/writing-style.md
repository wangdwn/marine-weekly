# 写作纪律与语义高亮协同

本刊的重点标注不由生成端负责：前端引擎用确定性正则对纯文本做加粗与数字高亮。
生成端的职责是**把真正重要的结论装进会被高亮的信号句式里**，把不重要的内容避开这些句式。
这是"提示词 × 渲染引擎"的协同设计，两端必须对齐。

## 一、输出形态

- 一律纯文本。禁止 `**加粗**`、`# 标题`、HTML 标签、Markdown 列表符。
  校验器会拦截 `**`、`##`、`<`、`>`。
- 段内换行用 `\n`（JSON 字符串内），前端转 `<br>`。少用，一个 content 通常一段。
- 句式短促：政务简报体，单句一般不超过 40 字；一个 content 控制在 60—140 字。

## 二、信号句式表（前端自动加粗的触发器）

| 用途 | 触发句式 | 使用纪律 |
|---|---|---|
| 定性判断 | `标志着…` `意味着…` | 只用于真正的转折/定性结论，每板块 ≤2 处 |
| 行动指向 | `建议…` `需关注…` `应重点…` `加快推动…` `抢抓…` `把握…` `宜尽快…` | 建议/summary 中使用；正文 content 不用 |
| 影响预判 | `将带动…` `将推动…` `将促进…` `将形成…` | 仅出现在 impact（[推断] 之后） |
| 首创性 | `首艘/首次/首个/首单/首批/首座/首台/首家/首宗`，可加 `全球/全国/国内/全省/全市` 前缀 | 必须有来源支撑，不得自封 |
| 里程碑 | `创历史新高` `历史性突破` `填补空白` `稳居` `全面突破` | 慎用，一期全刊 ≤3 处 |
| 节点事件 | `正式印发` `正式施行` `获批` `全面开工` `竣工验收` `完成试航` `全面启动` `正式落地` | 事实性节点，用于 content |
| 行情信号 | `量价齐升` `超级周期` `景气度高` `持续高位` `高位运行` `大幅反弹` | 仅在有数据支撑时用 |
| 结构标记 | `热点一：` `热点二：` `热点三：` | 仅用于 overview.summary |

数字高亮：任何 `数字+单位` 会被自动高亮（元、亿元、万元、元/吨、吨、万吨、艘、座、
千米、公里、平方公里、公顷、标准箱、万标箱、万人次、亿千瓦时、万千瓦、兆瓦、%、
个百分点、倍、点、家、项、宗、个、班次 等）。因此：**关键数字务必带单位写全**；
不想被强调的次要数字，改为定性表述（如"逾百座"）。

## 三、事实 / 研判分离（最重要的一条）

- `content` = 事实层。只写可溯源、可核验的事实：谁、何时、发生了什么、规模多少。
  不得出现"将""预计""可能""有望"引导的预测，不得出现评价性形容词。
- `impact` = 研判层。必须以 `[推断] ` 开头（前端渲染为独立标记）。研判要回答
  "这对我们意味着什么"，并尽量指向可操作的方向。
- 一条 item 只允许一个 `[推断]`，且只在 impact 字段。content、summary、source
  中出现 `[推断]` 视为违规。

## 四、来源纪律

- `source` 格式：`来源：机构或文件名，M月D日`。多来源用分号分隔。
- 数字必须能落到 source；落不到的数字降级为定性描述或删除。
- 来源不确定时如实写"待核实"并在交付说明中向用户点出，不得虚构机构名。

## 五、工作建议的质量门槛

每条 suggest 必须同时回答三问，缺一不可：

1. **做什么**：具体动作（对接谁、编制什么、启动什么），动宾结构开头。
2. **为什么是现在**：时间窗口依据（写入 body 或 basis）。
3. **依据是什么**：`basis` 回指本期具体板块的具体事实（格式：`本期××板块：…`）。

反例（禁止）："建议加大蓝碳工作力度，积极对接相关部门，持续关注政策动向。"
正例："建议梳理本轮竞配中标业主清单，主动对接其深远海场址前期勘察需求，
两周内形成标准化服务包介绍材料。"

## 六、语体基调

- 内参体：客观、克制、短句、不用感叹号，不用营销化形容词（"火爆""重磅""震撼"）。
- 涨跌表述遵循国内惯例：红涨绿跌由前端配色，文字端确保 `isUp` 与 `change` 方向一致。
- 全刊自称"本报""本院"；对上级机构使用全称或规范简称。
